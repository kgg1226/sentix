## Description

pipeline-worker.js의 shell:true로 인한 plan phase 실패 - spawnSync/spawn에서 shell:true 사용 시 프롬프트 내 괄호 등 특수문자가 쉘에 의해 해석되어 exit code 2 발생. shell:false로 변경 필요

## Root Cause Analysis

bug-008 수정에서 Windows EPERM 회피를 위해 `shell: true` 를 하드코딩했다.
Windows 의 .cmd 런처는 `shell: true` 없이 직접 spawn 하면 EPERM 을 발생시키지만,
Unix 계열에서는 `shell: true` 를 쓰면 `/bin/sh -c "claude -p ..."` 로 실행되어
프롬프트 내 괄호 `()`, 세미콜론 `;`, 백틱 등이 쉘에 의해 재해석된다.
plan phase 프롬프트에 괄호가 포함된 설명이 들어가면 sh 가 서브쉘로 해석해
구문 오류 → exit code 2 로 실패.

## Resolution

`const USE_SHELL = process.platform === "win32"` 를 도입해 플랫폼별 분기:
- Windows: `shell: true` 유지 (bug-008 회귀 방지)
- Unix: `shell: false` 로 특수문자 안전 처리 (bug-010 해결)

`src/lib/pipeline-worker.js` 의 `runPhase` (spawnSync) 와 `spawnWorker` (spawn)
양쪽 모두 `shell: USE_SHELL` 로 교체. 기존 export/기능/테스트 변경 없음.
