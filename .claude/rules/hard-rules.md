# Hard Rules — 코드 수정 시 필수 준수

1. **테스트 스냅샷 필수**: 코드 수정 전 `npm test` 실행하여 기준선 확보
2. **SCOPE 밖 수정 금지**: 티켓에 정의된 파일만 수정. 범위 밖이면 Governor에게 "SCOPE 확장 필요" 반환
3. **export/API 삭제 금지**: 기존 export를 삭제하면 의존하는 모듈이 깨짐. 시그니처 변경 불가피 시 planner 재소환
4. **테스트 삭제/약화 금지**: 테스트가 실패하면 코드를 고친다, 테스트를 고치지 않는다
5. **순삭제 50줄 제한**: 초과 시 리팩터링을 별도 티켓으로 분리
6. **기능/핸들러 삭제 금지**: 버그가 있는 기능은 고치는 것이지 없애는 것이 아니다
